    üéµ Music Player System Design¬∂

    1. Overview¬∂

    A scalable and modular music player system that allows users to play, pause, and manage songs, create playlists, and play them in different modes. The system supports multiple output devices and is designed for easy feature extension.

    2. Functional Requirements¬∂

    üéß Core Features¬∂

    1. Play / Pause Song
    2. User can play or pause any selected song.

    3. Support for resuming from the paused position.

    4. Playlist Management

    5. Users can create multiple playlists.
    6. Add or remove songs from playlists.

    7. Rename or delete playlists.

    8. Playback Modes

    9. Sequential Mode ‚Äì Songs play in order.
    10. Shuffle Mode ‚Äì Songs play in random order.

    11. Custom Mode ‚Äì Users can define their own play order.

    12. Multi-Device Audio Output

    13. Supports Bluetooth speakers, wired speakers, and headphones.
    14. Allows switching between devices seamlessly during playback.

    3. Non-Functional Requirements¬∂

    ‚öôÔ∏è Scalability¬∂

    • The architecture should allow easy scaling to handle more users, songs, or playlists.
    • Use modular components (Player, PlaylistManager, DeviceManager, etc.) to enable distributed scaling.

    üß© Extensibility¬∂

    • New features (e.g., new playback modes or output devices) should be easily integrated without modifying core logic.
    • Follow Open/Closed Principle (OCP) for code design ‚Äî open for extension, closed for modification.

    üíª Performance¬∂

    • Ensure low latency in playback and fast switching between songs or devices.

    üß† Maintainability¬∂

    • Use clear interfaces and design patterns (e.g., Strategy Pattern for playback modes, Observer Pattern for device state changes).

    4. High-Level Design¬∂

    üèóÔ∏è System Components¬∂

    Component Responsibility
    Song Manager Handles song metadata, loading, and caching.
    Player Controller Controls play, pause, resume, and stop operations.
    Playlist Manager Manages playlists and song sequencing.
    Playback Strategy Defines different playback modes (Sequential, Shuffle, Custom).
    Device Manager Detects and manages available output devices.
    User Interface Allows user interaction and displays playback status.

    5. Design Patterns Used¬∂

    • Strategy Pattern ‚Üí For flexible playback modes.
    • Observer Pattern ‚Üí For device connection/disconnection updates.
    • Factory Pattern ‚Üí For creating device objects dynamically.
    • Facade Pattern ‚Üí For providing a unified interface to core subsystems.

    6. Future Extensions¬∂

    • Integration with cloud storage for songs.
    • Smart playlist recommendations using AI.
    • Voice command support for controlling playback.
    • Cross-platform support (web, mobile, desktop).

    7. Example Flow¬∂

    1. User opens app ‚Üí selects playlist.
    2. System loads playlist ‚Üí applies selected playback strategy.
    3. User selects output device ‚Üí playback starts.
    4. User can pause/resume or switch device anytime.

 20struct Song {
 21    int id;
 22    std::string title;
 23    int durationSeconds; // fake metadata

 26// Simple SongManager: stores songs and returns metadata
 27class SongManager {
 28public:
 29    void addSong(const Song &s) { songs_[s.id] = s; }
 30    Song getSong(int id) const {
 31        auto it = songs_.find(id);
 32        if (it != songs_.end()) return it->second;
 33        throw std::runtime_error("Song not found");
 34    }
 35private:
 36    std::unordered_map<int, Song> songs_;
 37};

 39// Playlist container
 40class Playlist {
 41public:
 42    explicit Playlist(std::string name) : name_(std::move(name)) {}
 43    void addSongId(int songId) { songIds_.push_back(songId); }
 44    void removeSongId(int songId) {
 45        songIds_.erase(std::remove(songIds_.begin(), songIds_.end(), songId), songIds_.end());
 46    }
 47    const std::vector<int>& getSongIds() const { return songIds_; }
 48    const std::string& getName() const { return name_; }
 49private:
 50    std::string name_;
 51    std::vector<int> songIds_;
 52};

 54class PlaylistManager {
 55public:
 56    std::shared_ptr<Playlist> createPlaylist(const std::string &name) {
 57        auto p = std::make_shared<Playlist>(name);
 58        playlists_.push_back(p);
 59        return p;
 60    }
 61    void deletePlaylist(const std::string &name) {
 62        playlists_.erase(std::remove_if(playlists_.begin(), playlists_.end(), [&](const auto &p){ return p->getName()==name; }), playlists_.end());
 63    }
 64private:
 65    std::vector<std::shared_ptr<Playlist>> playlists_;
 66};

126// PlaybackStrategy interface
127class PlaybackStrategy {
128public:
129    virtual ~PlaybackStrategy() = default;
130    virtual void setPlaylist(const std::vector<int> &songIds) = 0;
131    virtual int getCurrentSongId() = 0; // returns -1 if none
132    virtual int next() = 0; // advances and returns new song id or -1
133    virtual int previous() = 0;
134    virtual void reset() = 0;
135};
136
137// Sequential playback
138class SequentialPlayback : public PlaybackStrategy {
139public:
140    void setPlaylist(const std::vector<int> &songIds) override { order_ = songIds; pos_ = 0; }
141    int getCurrentSongId() override { if (order_.empty() || pos_ >= order_.size()) return -1; return order_[pos_]; }
142    int next() override { if (pos_ + 1 < order_.size()) ++pos_; else pos_ = order_.size(); return getCurrentSongId(); }
143    int previous() override { if (pos_ == 0) return getCurrentSongId(); --pos_; return getCurrentSongId(); }
144    void reset() override { pos_ = 0; }
145private:
146    std::vector<int> order_;
147    size_t pos_ = 0;
148};
149
150// Shuffle playback
151class ShufflePlayback : public PlaybackStrategy {
152public:
153    void setPlaylist(const std::vector<int> &songIds) override {
154        original_ = songIds;
155        order_ = songIds;
156        std::random_device rd;
157        std::mt19937 g(rd());
158        std::shuffle(order_.begin(), order_.end(), g);
159        pos_ = 0;
160    }
161    int getCurrentSongId() override { if (order_.empty() || pos_ >= order_.size()) return -1; return order_[pos_]; }
162    int next() override { if (pos_ + 1 < order_.size()) ++pos_; else pos_ = order_.size(); return getCurrentSongId(); }
163    int previous() override { if (pos_ == 0) return getCurrentSongId(); --pos_; return getCurrentSongId(); }
164    void reset() override { pos_ = 0; std::random_device rd; std::mt19937 g(rd()); std::shuffle(order_.begin(), order_.end(), g); }
165private:
166    std::vector<int> original_;
167    std::vector<int> order_;
168    size_t pos_ = 0;
169};
170
171// Custom playback: user provides a custom order
172class CustomPlayback : public PlaybackStrategy {
173public:
174    explicit CustomPlayback(const std::vector<int> &customOrder = {}) { order_ = customOrder; pos_ = 0; }
175    void setPlaylist(const std::vector<int> &songIds) override {
176        // If custom order was given and matches the songIds set, keep it. Otherwise default to given list.
177        // For simplicity we default to songIds if custom order is empty.
178        if (order_.empty()) { order_ = songIds; }
179        pos_ = 0;
180    }
181    int getCurrentSongId() override { if (order_.empty() || pos_ >= order_.size()) return -1; return order_[pos_]; }
182    int next() override { if (pos_ + 1 < order_.size()) ++pos_; else pos_ = order_.size(); return getCurrentSongId(); }
183    int previous() override { if (pos_ == 0) return getCurrentSongId(); --pos_; return getCurrentSongId(); }
184    void reset() override { pos_ = 0; }
185private:
186    std::vector<int> order_;
187    size_t pos_ = 0;
188};
189
190// PlayerController: orchestrates playback
191class PlayerController {
192public:
193    PlayerController(const SongManager &sm, DeviceManager &dm) : songManager_(sm), deviceManager_(dm) {}
194
195    void setPlaybackStrategy(std::unique_ptr<PlaybackStrategy> strat) {
196        std::lock_guard<std::mutex> lock(m_);
197        playbackStrategy_ = std::move(strat);
198        if (currentPlaylist_) playbackStrategy_->setPlaylist(currentPlaylist_->getSongIds());
199    }
200
201    void loadPlaylist(std::shared_ptr<Playlist> p) {
202        std::lock_guard<std::mutex> lock(m_);
203        currentPlaylist_ = std::move(p);
204        if (playbackStrategy_) playbackStrategy_->setPlaylist(currentPlaylist_->getSongIds());
205    }
206
207    void play() {
208        std::lock_guard<std::mutex> lock(m_);
209        if (!playbackStrategy_) { std::cerr << "No playback strategy set\n"; return; }
210        if (!deviceManager_.getActiveDevice()) { std::cerr << "No active device selected\n"; return; }
211        int id = playbackStrategy_->getCurrentSongId();
212        if (id == -1) { std::cerr << "Nothing to play\n"; return; }
213        Song s = songManager_.getSong(id);
214        playing_ = true;
215        std::cout << "-> Playing: " << s.title << "\n";
216        deviceManager_.getActiveDevice()->playAudio(s);
217        // NOTE: we do not actually stream audio; we simulate playing via thread sleep for demonstration
218        playThread_ = std::thread([this, s]() {
219            std::this_thread::sleep_for(std::chrono::milliseconds(std::min(2000, s.durationSeconds*1000)));
220            // when finished, auto-advance
221            std::lock_guard<std::mutex> lock(m_);
222            if (playing_) {
223                int nextId = playbackStrategy_->next();
224                if (nextId != -1) {
225                    // play next automatically
226                    Song nextSong = songManager_.getSong(nextId);
227                    std::cout << "-> Auto-advancing to: " << nextSong.title << "\n";
228                    deviceManager_.getActiveDevice()->playAudio(nextSong);
229                } else {
230                    std::cout << "Playlist finished\n";
231                    deviceManager_.getActiveDevice()->stopAudio();
232                    playing_ = false;
233                }
234            }
235        });
236        if (playThread_.joinable()) playThread_.detach();
237    }
238
239    void pause() {
240        std::lock_guard<std::mutex> lock(m_);
241        if (!playing_) { std::cerr << "Nothing is playing\n"; return; }
242        playing_ = false;
243        if (deviceManager_.getActiveDevice()) deviceManager_.getActiveDevice()->stopAudio();
244        std::cout << "Paused\n";
245    }
246
247    void next() {
248        std::lock_guard<std::mutex> lock(m_);
249        if (!playbackStrategy_) return;
250        int id = playbackStrategy_->next();
251        if (id == -1) { std::cout << "End of playlist\n"; return; }
252        Song s = songManager_.getSong(id);
253        if (deviceManager_.getActiveDevice()) deviceManager_.getActiveDevice()->playAudio(s);
254        playing_ = true;
255    }
256
257    void previous() {
258        std::lock_guard<std::mutex> lock(m_);
259        if (!playbackStrategy_) return;
260        int id = playbackStrategy_->previous();
261        if (id == -1) return;
262        Song s = songManager_.getSong(id);
263        if (deviceManager_.getActiveDevice()) deviceManager_.getActiveDevice()->playAudio(s);
264        playing_ = true;
265    }
266
267private:
268    const SongManager &songManager_;
269    DeviceManager &deviceManager_;
270    std::unique_ptr<PlaybackStrategy> playbackStrategy_;
271    std::shared_ptr<Playlist> currentPlaylist_;
272    bool playing_ = false;
273    std::thread playThread_;
274    std::mutex m_;
275};

294int main() {
295    using namespace MusicPlayer;
296
297    SongManager songManager;
298    songManager.addSong({1, "Song A", 3});
299    songManager.addSong({2, "Song B", 4});
300    songManager.addSong({3, "Song C", 5});
301
302    PlaylistManager playlistManager;
303    auto pl = playlistManager.createPlaylist("My Favs");
304    pl->addSongId(1);
305    pl->addSongId(2);
306    pl->addSongId(3);
307
308    DeviceManager deviceManager;
309    deviceManager.addDevice(std::make_shared<BluetoothSpeaker>());
310    deviceManager.addDevice(std::make_shared<WiredSpeaker>());
311    deviceManager.addDevice(std::make_shared<Headphones>());
312    deviceManager.listDevices();
313    deviceManager.switchTo(0); // bluetooth
314
315    PlayerController controller(songManager, deviceManager);
316
317    // Sequential playback
318    controller.setPlaybackStrategy(std::make_unique<SequentialPlayback>());
319    controller.loadPlaylist(pl);
320
321    controller.play();
322    std::this_thread::sleep_for(std::chrono::seconds(1));
323    controller.pause();
324
325    // switch device while paused
326    deviceManager.switchTo(2); // headphones
327
328    controller.play();
329
330    // demonstrate shuffle
331    controller.setPlaybackStrategy(std::make_unique<ShufflePlayback>());
332    controller.loadPlaylist(pl);
333    controller.play();
334
335    // keep main alive briefly to let threads print
336    std::this_thread::sleep_for(std::chrono::seconds(3));
337    return 0;
338}

    This design focuses on modularity, scalability, and extensibility ‚Äî allowing easy integration of new features (like new devices or play modes) without reworking the core architecture.
