    ‚ôüÔ∏è Multiplayer Chess System Design¬∂

    Overview¬∂

    The Multiplayer Chess Platform enables multiple users to play chess simultaneously under standard rules, with features like score-based matchmaking, in-game chat, and session handling when users quit or finish a game.

    The system is designed to be scalable, extensible, and real-time for competitive gameplay and future enhancements.

    üéÆ Functional Requirements¬∂

    1. Multiple Concurrent Matches

    • Multiple users can play chess simultaneously in different sessions.
    • Each match runs independently, allowing scalability.

    2. Score-Based Matchmaking

    • Users are paired based on similar skill or rating (ELO-based or score-based algorithm).
    • The matchmaking system dynamically searches for opponents within a score tolerance range.

    3. Standard Chess Rules (Scalable)

    • The game engine enforces standard chess rules such as piece movement, captures, and turns.
    • The system design allows easy extension to add features like castling, pawn promotion, timers, and new variants (e.g., blitz chess, 3-check).

    4. In-Match Chat System

    • Players can send and receive messages during a match.
    • Chat is isolated to each game and uses a thread-safe message queue.
    • Chat logs can be optionally stored for replay or moderation.

    5. User Quit Handling

    • Players can quit a match at any time.
    • The session manager detects early quits and declares the other player as the winner or records the match as incomplete.
    • Resources and match states are cleaned up gracefully.

    üß© System Components¬∂

    1. User Management

    • Handles user registration, login, and profile management.
    • Tracks player ID, username, and rating.
    • Maintains a list of active users and their session states.
    • Supports concurrent user sessions safely.

    Responsibilities:

    • Register new users.
    • Retrieve player information.
    • Manage user activity status.

    2. Matchmaking System

    • Implements a score-based pairing algorithm that matches players with similar ELO or performance scores.
    • Supports dynamic queues for users waiting for matches.
    • Automatically initiates matches once suitable pairs are found.

    Responsibilities:

    • Maintain a queue of waiting players.
    • Pair players within a score range (e.g., ¬±200).
    • Initiate new game sessions.
    • Clean up finished or disconnected matches.

    3. Game Engine

    • Core logic for chess gameplay.
    • Responsible for move validation, piece interactions, and maintaining the board state.

    Responsibilities:

    • Implement movement rules for all pieces.
    • Check for valid moves and enforce turns.
    • Handle pawn promotion, captures, and king safety.
    • Detect match end conditions (checkmate, stalemate, or resignation).

    Extensibility:

    • Support for timers, undo moves, or AI based analysis.
    • Add new chess variants (e.g., 3-player chess or timed modes).

    4. Chat System

    • Enables real-time communication between players during a match.
    • Each match maintains its own message queue.
    • Thread-safe design ensures consistency under concurrent message sends.

    Responsibilities:

    • Send and receive messages.
    • Maintain per match chat history.
    • Optionally integrate moderation and filtering.

    5. Session Management

    • Oversees match lifecycle, including start, progress, and termination.
    • Monitors player activity (e.g., quit, timeout, or disconnect).
    • Ensures resources are released and results are saved properly.

    Responsibilities:

    • Detect if a player quits mid-game.
    • End match gracefully and update user stats.
    • Save match results and logs.
    • Manage ongoing match threads and cleanup.

    System Flow¬∂

    1. User Registration/Login

    • User connects to the platform and logs in.
    • User is added to the pool of active players.

    2. Matchmaking Process

    • User requests a match.
    • The matchmaking system finds an opponent within a score range.
    • A new match session is created.

    3. Game Execution

    • Game Engine initializes a new board.
    • Players alternate turns making valid moves.
    • Real-time updates are broadcast to both players.

    4. In-Match Chat

    • Players exchange messages through the chat system.
    • Messages are processed asynchronously.

    5. Session Termination

    • Match ends via checkmate, draw, timeout, or user quit.
    • Results and chat history are logged.
    • Players can re-enter the matchmaking pool.

    Scalability & Extensibility¬∂

    Scalability:

    • Designed to support hundreds of concurrent matches using independent threads or asynchronous I/O (in production servers, this would use event-driven architecture).

    Extensibility:

    • Easy to integrate:
    • New matchmaking logic (league, tournament).
    • AI opponents (Stockfish or custom engine).
    • New UIs or APIs via REST/WebSocket.

    Resilience:

    • Handles early quits, dropped connections, and concurrency safely.

 27/* -----------------------------
 28   Thread-safe Queue for Chat
 29   ----------------------------- */
 30template<typename T>
 31class TSQueue {
 32    mutex m;
 33    condition_variable cv;
 34    deque<T> q;
 35public:
 36    void push(const T& v) {
 37        {
 38            lock_guard<mutex> lk(m);
 39            q.push_back(v);
 40        }
 41        cv.notify_one();
 42    }
 43    // wait and pop one
 44    T wait_pop() {
 45        unique_lock<mutex> lk(m);
 46        cv.wait(lk, [&]{ return !q.empty(); });
 47        T v = q.front(); q.pop_front();
 48        return v;
 49    }
 50    bool try_pop(T &out) {
 51        lock_guard<mutex> lk(m);
 52        if(q.empty()) return false;
 53        out = q.front(); q.pop_front();
 54        return true;
 55    }
 56    bool empty() {
 57        lock_guard<mutex> lk(m);
 58        return q.empty();
 59    }
 60};

 88/* -----------------------------
 89   Simple Chess Board & Engine
 90   - 8x8 board
 91   - Basic rules: piece movement checks (no castling/en-passant), simple checkmate detection is NOT complete.
 92   - For demonstration only.
 93   ----------------------------- */

 95struct Cell {
 96    Piece piece;
 97    Color color;
 98    Cell(): piece(Piece::EMPTY), color(Color::NONE) {}
 99    Cell(Piece p, Color c): piece(p), color(c) {}
100    bool empty() const { return piece == Piece::EMPTY; }
101};
102
103struct Pos {
104    int r, c;
105    Pos(int rr=0,int cc=0): r(rr), c(cc) {}
106    bool inBounds() const { return r>=0 && r<8 && c>=0 && c<8; }
107    bool operator==(const Pos& o) const { return r==o.r && c==o.c; }
108};
109
110string posToStr(const Pos& p) {
111    string s;
112    s += char('a' + p.c);
113    s += char('1' + (7 - p.r)); // row 0 is rank 8
114    return s;
115}
116
117class GameEngine {
118    array<array<Cell,8>,8> board;
119    Color turn;
120    atomic<bool> finished;
121    mutex m; // protect board + state
122
123public:
124    GameEngine() { reset(); }
125
126    void reset() {
127        lock_guard<mutex> lk(m);
128        // Clear
129        for (int i=0;i<8;i++) for (int j=0;j<8;j++) board[i][j]=Cell();
130        // Setup pawns
131        for (int j=0;j<8;j++) {
132            board[1][j] = Cell(Piece::P, Color::BLACK);
133            board[6][j] = Cell(Piece::P, Color::WHITE);
134        }
135        // Rooks
136        board[0][0]=board[0][7]=Cell(Piece::R, Color::BLACK);
137        board[7][0]=board[7][7]=Cell(Piece::R, Color::WHITE);
138        // Knights
139        board[0][1]=board[0][6]=Cell(Piece::N, Color::BLACK);
140        board[7][1]=board[7][6]=Cell(Piece::N, Color::WHITE);
141        // Bishops
142        board[0][2]=board[0][5]=Cell(Piece::B, Color::BLACK);
143        board[7][2]=board[7][5]=Cell(Piece::B, Color::WHITE);
144        // Queens
145        board[0][3]=Cell(Piece::Q, Color::BLACK);
146        board[7][3]=Cell(Piece::Q, Color::WHITE);
147        // Kings
148        board[0][4]=Cell(Piece::K, Color::BLACK);
149        board[7][4]=Cell(Piece::K, Color::WHITE);
150
151        turn = Color::WHITE;
152        finished = false;
153    }
154
155    // Very simple move parser from "e2e4" format
156    bool parsePos(const string &s, Pos &out) {
157        if (s.size() != 2) return false;
158        char file = s[0];
159        char rank = s[1];
160        if (file < 'a' || file > 'h') return false;
161        if (rank < '1' || rank > '8') return false;
162        int c = file - 'a';
163        int r = 7 - (rank - '1');
164        out = Pos(r,c);
165        return true;
166    }
167
168    // Basic move validation (not exhaustive)
169    bool validMove(const Pos &from, const Pos &to, Color player) {
170        if (!from.inBounds() || !to.inBounds()) return false;
171        Cell src = board[from.r][from.c];
172        Cell dst = board[to.r][to.c];
173        if (src.empty()) return false;
174        if (src.color != player) return false;
175        if (!dst.empty() && dst.color == player) return false;
176
177        int dr = to.r - from.r;
178        int dc = to.c - from.c;
179        Piece p = src.piece;
180        switch (p) {
181            case Piece::P: {
182                int dir = (src.color==Color::WHITE) ? -1 : 1;
183                // single step
184                if (dc == 0 && dr == dir && dst.empty()) return true;
185                // initial double step
186                if (dc == 0 && dr == 2*dir) {
187                    int startRow = (src.color==Color::WHITE) ? 6 : 1;
188                    if (from.r == startRow && dst.empty() && board[from.r+dir][from.c].empty()) return true;
189                }
190                // capture
191                if (abs(dc)==1 && dr==dir && !dst.empty() && dst.color != src.color) return true;
192                return false;
193            }
194            case Piece::R: {
195                if (dr!=0 && dc!=0) return false;
196                int rr = (dr==0) ? 0 : (dr>0?1:-1);
197                int cc = (dc==0) ? 0 : (dc>0?1:-1);
198                int steps = max(abs(dr),abs(dc));
199                for (int s=1;s<steps;s++) if (!board[from.r + rr*s][from.c + cc*s].empty()) return false;
200                return true;
201            }
202            case Piece::B: {
203                if (abs(dr) != abs(dc)) return false;
204                int rr = (dr>0?1:-1), cc=(dc>0?1:-1);
205                for (int s=1;s<abs(dr);s++) if (!board[from.r + rr*s][from.c + cc*s].empty()) return false;
206                return true;
207            }
208            case Piece::Q: {
209                if (dr==0 || dc==0) {
210                    int rr = (dr==0)?0:(dr>0?1:-1);
211                    int cc = (dc==0)?0:(dc>0?1:-1);
212                    int steps = max(abs(dr),abs(dc));
213                    for (int s=1;s<steps;s++) if (!board[from.r + rr*s][from.c + cc*s].empty()) return false;
214                    return true;
215                } else if (abs(dr)==abs(dc)) {
216                    int rr=(dr>0?1:-1), cc=(dc>0?1:-1);
217                    for (int s=1;s<abs(dr);s++) if (!board[from.r + rr*s][from.c + cc*s].empty()) return false;
218                    return true;
219                } else return false;
220            }
221            case Piece::N: {
222                if ((abs(dr)==2 && abs(dc)==1) || (abs(dr)==1 && abs(dc)==2)) return true;
223                return false;
224            }
225            case Piece::K: {
226                if (max(abs(dr),abs(dc))==1) return true;
227                return false;
228            }
229            default: return false;
230        }
231    }
232
233    // Move piece, return true if moved
234    bool makeMove(const string &fromStr, const string &toStr, Color player, string &err) {
235        lock_guard<mutex> lk(m);
236        if (finished) { err = "Game already finished"; return false; }
237        Pos from, to;
238        if (!parsePos(fromStr, from) || !parsePos(toStr, to)) { err = "Invalid coordinates"; return false; }
239        if (!validMove(from, to, player)) { err = "Illegal move"; return false; }
240        // perform
241        Cell src = board[from.r][from.c];
242        Cell dst = board[to.r][to.c];
243        board[to.r][to.c] = src;
244        board[from.r][from.c] = Cell();
245        // pawn promotion simplification: promote to queen if reaches last rank
246        if (src.piece == Piece::P) {
247            if ((src.color==Color::WHITE && to.r==0) || (src.color==Color::BLACK && to.r==7)) {
248                board[to.r][to.c].piece = Piece::Q;
249            }
250        }
251        // swap turn
252        turn = (turn==Color::WHITE?Color::BLACK:Color::WHITE);
253        return true;
254    }
255
256    // Very simple "is game over?" - here we just detect if a king is captured.
257    bool checkFinished() {
258        lock_guard<mutex> lk(m);
259        bool whiteKing=false, blackKing=false;
260        for (int i=0;i<8;i++) for (int j=0;j<8;j++) {
261            if (board[i][j].piece == Piece::K) {
262                if (board[i][j].color == Color::WHITE) whiteKing=true;
263                if (board[i][j].color == Color::BLACK) blackKing=true;
264            }
265        }
266        if (!whiteKing || !blackKing) {
267            finished = true;
268            return true;
269        }
270        return false;
271    }
272
273    // Return FEN-like simple board print
274    string boardStr() {
275        lock_guard<mutex> lk(m);
276        string s;
277        for (int i=0;i<8;i++) {
278            for (int j=0;j<8;j++) {
279                Cell c = board[i][j];
280                char ch='.';
281                if (!c.empty()) {
282                    switch (c.piece) {
283                        case Piece::P: ch='p'; break;
284                        case Piece::R: ch='r'; break;
285                        case Piece::N: ch='n'; break;
286                        case Piece::B: ch='b'; break;
287                        case Piece::Q: ch='q'; break;
288                        case Piece::K: ch='k'; break;
289                        default: ch='?';
290                    }
291                    if (c.color==Color::WHITE) ch = toupper(ch);
292                }
293                s += ch; s += ' ';
294            }
295            s += '\n';
296        }
297        return s;
298    }
299
300    Color currentTurn() {
301        lock_guard<mutex> lk(m);
302        return turn;
303    }
304
305    void forceFinish() {
306        finished = true;
307    }
308};

310/* -----------------------------

313struct ChatMessage {
314    UserId from;
315    string text;
316    chrono::system_clock::time_point ts;
317};

319class ChatSystem {
320    TSQueue<ChatMessage> q;
321public:
322    void send(UserId from, const string &text) {
323        q.push(ChatMessage{from, text, chrono::system_clock::now()});
324    }
325    // blocking receive (simulate pushing to client)
326    ChatMessage receiveBlocking() {
327        return q.wait_pop();
328    }
329    // try non-blocking
330    bool tryReceive(ChatMessage &out) {
331        return q.try_pop(out);
332    }
333};

335/* -----------------------------
336   Match / Session Management
337   ----------------------------- */

339class Match {
340    MatchId id;
341    PlayerInfo whitePlayer;
342    PlayerInfo blackPlayer;
343    GameEngine game;
344    ChatSystem chat;
345    thread matchThread;
346    atomic<bool> running;
347    mutex outm; // printing
348    atomic<bool> whiteQuit, blackQuit;
349
350public:
351    Match(MatchId mid, PlayerInfo p1, PlayerInfo p2)
352    : id(mid), whitePlayer(p1), blackPlayer(p2), running(false), whiteQuit(false), blackQuit(false)
353    {
354        // We assign higher rated player to white arbitrarily
355        if (whitePlayer.score < blackPlayer.score) swap(whitePlayer, blackPlayer);
356    }
357
358    void start() {
359        running = true;
360        matchThread = thread(&Match::run, this);
361    }
362
363    void joinThread() {
364        if (matchThread.joinable()) matchThread.join();
365    }
366
367    void run() {
368        {
369            lock_guard<mutex> lk(outm);
370            cout << "[Match " << id << "] Started: " << whitePlayer.name << " (white) vs " << blackPlayer.name << " (black)\n";
371        }
372        game.reset();
373        // Simulated play loop: to show functioning, we will alternate "random moves" for a few turns or accept quittings
374        // In a real system we would accept moves from networked players.
375        int turnCount = 0;
376        while (running) {
377            if (whiteQuit || blackQuit) {
378                lock_guard<mutex> lk(outm);
379                cout << "[Match " << id << "] A player quit. ";
380                if (whiteQuit) cout << whitePlayer.name << " quit. ";
381                if (blackQuit) cout << blackPlayer.name << " quit. ";
382                cout << "Terminating match.\n";
383                game.forceFinish();
384                break;
385            }
386            // For demo: process chat messages if any
387            ChatMessage cm;
388            while (chat.tryReceive(cm)) {
389                lock_guard<mutex> lk(outm);
390                string who = (cm.from == whitePlayer.id) ? whitePlayer.name : (cm.from == blackPlayer.id ? blackPlayer.name : "unknown");
391                cout << "[Match " << id << "][CHAT] " << who << ": " << cm.text << "\n";
392            }
393
394            // Check finished
395            if (game.checkFinished()) {
396                lock_guard<mutex> lk(outm);
397                cout << "[Match " << id << "] Game finished by capture of king.\n";
398                break;
399            }
400
401            // For simulation, make a dummy legal move for current player from a list of candidate moves.
402            Color cur = game.currentTurn();
403            PlayerInfo curPlayer = (cur==Color::WHITE) ? whitePlayer : blackPlayer;
404            // simulate quitting: if random small chance player quits
405            if ((rand()%1000) < 5) { // tiny chance
406                if (cur==Color::WHITE) whiteQuit = true;
407                else blackQuit = true;
408                continue;
409            }
410
411            // We will search simple moves: move any pawn forward if possible, else try small moves
412            bool moved = false;
413            // trying to find any legal move brute-force
414            for (int r=0;r<8 && !moved;r++) for (int c=0;c<8 && !moved;c++) {
415                Pos from(r,c);
416                // build coordinate strings
417                // attempt to move to all possible squares
418                for (int rr=0;rr<8 && !moved;rr++) for (int cc=0;cc<8 && !moved;cc++) {
419                    Pos to(rr,cc);
420                    // skip same
421                    if (from==to) continue;
422                    // prepare strings
423                    string fromS = posToStr(from);
424                    string toS = posToStr(to);
425                    string err;
426                    if (game.makeMove(fromS, toS, cur, err)) {
427                        // succeeded
428                        lock_guard<mutex> lk(outm);
429                        cout << "[Match " << id << "] " << curPlayer.name << " (" << (cur==Color::WHITE?"W":"B") << ") moved " << fromS << " -> " << toS << "\n";
430                        moved = true;
431                        break;
432                    }
433                }
434            }
435            // if no move found (stalemate or blocked) ‚Äî finish
436            if (!moved) {
437                lock_guard<mutex> lk(outm);
438                cout << "[Match " << id << "] No legal move found for current player. Ending match.\n";
439                break;
440            }
441
442            turnCount++;
443            if (turnCount > 200) { // limit for demo
444                lock_guard<mutex> lk(outm);
445                cout << "[Match " << id << "] Move limit reached, ending match as draw.\n";
446                break;
447            }
448
449            // small sleep to let output be readable
450            this_thread::sleep_for(chrono::milliseconds(50));
451        } // end while
452
453        {
454            lock_guard<mutex> lk(outm);
455            cout << "[Match " << id << "] cleaning up\n";
456        }
457        running = false;
458    }
459
460    void sendChat(UserId from, const string &text) {
461        chat.send(from, text);
462    }
463
464    void playerQuit(UserId uid) {
465        if (uid == whitePlayer.id) whiteQuit = true;
466        else if (uid == blackPlayer.id) blackQuit = true;
467    }
468
469    bool isRunning() const { return running; }
470};

472/* -----------------------------
473   MatchMaker
474   - Simple queue-based matchmaking by score proximity
475   ----------------------------- */
476
477class MatchMaker {
478    mutex m;
479    MatchId nextMatchId = 1;
480    vector<shared_ptr<Match>> matches;
481    deque<PlayerInfo> waiting;
482    int scoreTolerance = 200; // +- rating window
483public:
484    // add waiting player and attempt to create matches
485    void enterQueue(const PlayerInfo &p) {
486        lock_guard<mutex> lk(m);
487        waiting.push_back(p);
488        tryMakeMatches();
489    }
490
491    // naive matching: scan waiting for pairs within tolerance
492    void tryMakeMatches() {
493        for (size_t i=0;i<waiting.size();++i) {
494            for (size_t j=i+1;j<waiting.size();++j) {
495                if (abs(waiting[i].score - waiting[j].score) <= scoreTolerance) {
496                    // create match
497                    PlayerInfo a = waiting[i];
498                    PlayerInfo b = waiting[j];
499                    // remove j then i
500                    waiting.erase(waiting.begin()+j);
501                    waiting.erase(waiting.begin()+i);
502                    makeMatch(a,b);
503                    // restart scanning
504                    tryMakeMatches();
505                    return;
506                }
507            }
508        }
509    }
510
511    void makeMatch(const PlayerInfo &a, const PlayerInfo &b) {
512        MatchId mid = nextMatchId++;
513        auto match = make_shared<Match>(mid, a, b);
514        matches.push_back(match);
515        match->start();
516    }
517
518    vector<shared_ptr<Match>> currentMatches() {
519        lock_guard<mutex> lk(m);
520        return matches;
521    }
522
523    // for demo: cleanup finished matches
524    void cleanup() {
525        lock_guard<mutex> lk(m);
526        vector<shared_ptr<Match>> active;
527        for (auto &mm : matches) {
528            if (mm->isRunning()) active.push_back(mm);
529            else mm->joinThread();
530        }
531        matches.swap(active);
532    }
533};
