    üßæ Payment Gateway System Design¬∂

    üìò Overview¬∂

    The Payment Gateway System is designed to support multiple payment providers such as Paytm, Razorpay, and others, following a standardized payment flow. It ensures scalability, robust error handling, and retry mechanisms, with the ability to easily integrate new payment gateways and recurring payment flows in the future.

    üéØ Design Goals¬∂

    1. Multi-Provider Support

    • Integrate multiple payment gateways (e.g., Paytm, Razorpay, Stripe, etc.).
    • Allow new providers to be added seamlessly without modifying the core logic.

    2. Standardized Payment Flow

    • All gateways follow a unified interface and workflow.
    • Includes validations, retries, and notification mechanisms.

    3. Error Handling & Retry Mechanism

    • Automatically handle transient errors with configurable retry strategies.
    • Supports multiple retry types (immediate, scheduled, exponential backoff).

    4. Extensibility

    • Easily add new gateways through a plug-and-play design using the Adapter pattern.

    5. Recurring Payments

    • Support automated recurring transactions using schedulers or cron-based triggers.

    üîÅ Standardized Payment Flow

    1. User initiates payment ‚Üí through frontend / API.

    2. Validation ‚Üí amount, currency, and provider.

    3. Select gateway ‚Üí based on provider name or configuration.

    4. Execute payment ‚Üí through the unified interface.

    5. Check response:

    • If success ‚Üí mark transaction as SUCCESS.
    • If failure ‚Üí invoke retry mechanism.

    6. Retry logic executes until:

    • Success achieved.
    • Retry attempts exhausted.

    7. Update transaction status in DB.

    8. Send notification to the user.

    9. If recurring payment enabled ‚Üí schedule next cycle.

    ‚öôÔ∏è Retry Strategies¬∂

    Type Description Example Use Case
    Immediate Retry Retry instantly after a failure (up to N attempts). Temporary network glitch.
    Scheduled Retry Retry after a fixed interval (e.g., every 5 mins). Payment gateway downtime.
    Exponential Backoff Retry with exponentially increasing delays (e.g., 1s ‚Üí 2s ‚Üí 4s). Unstable gateway response or network latency.

    üß† Design Patterns Used¬∂

    Pattern Purpose
    Adapter Pattern To integrate multiple payment gateways with a uniform interface.
    Strategy Pattern To allow flexible retry mechanisms without changing core logic.
    Observer Pattern (Notification System) To notify users asynchronously after payment events.
    Template Method Pattern To define a standard payment flow with overridable steps for different providers.

 12/* -------------------- Basic Types -------------------- */
 13
 14enum class PaymentStatus { SUCCESS, FAILED, PENDING };
 15struct PaymentRequest {
 16    string id;
 17    string provider;     // "paytm", "razorpay", etc.
 18    double amount;
 19    string currency;
 20    string userId;
 21    bool recurring = false;
 22    int remainingCycles = 0; // used for recurring demo
 23};
 24
 25struct PaymentResponse {
 26    PaymentStatus status;
 27    string providerTxId;
 28    string message;
 29};

 31/* -------------------- Payment Gateway Interface & Adapters -------------------- */
 32
 33class PaymentGateway {
 34public:
 35    virtual PaymentResponse process(const PaymentRequest &req) = 0;
 36    virtual string name() const = 0;
 37    virtual ~PaymentGateway() = default;
 38};
 39
 40// Simulate randomness for provider success/failure
 41static std::mt19937_64 rng((uint64_t)std::chrono::high_resolution_clock::now().time_since_epoch().count());
 42
 43class PaytmAdapter : public PaymentGateway {
 44public:
 45    PaymentResponse process(const PaymentRequest &req) override {
 46        // Simulate a 75% success rate
 47        std::uniform_real_distribution<double> dist(0.0, 1.0);
 48        double v = dist(rng);
 49        if (v < 0.75) {
 50            return {PaymentStatus::SUCCESS, "PAYTM-" + req.id + "-" + std::to_string((int)(v*10000)), "OK"};
 51        } else {
 52            return {PaymentStatus::FAILED, "", "Paytm: transient error"};
 53        }
 54    }
 55    string name() const override { return "paytm"; }
 56};
 57
 58class RazorpayAdapter : public PaymentGateway {
 59public:
 60    PaymentResponse process(const PaymentRequest &req) override {
 61        // Simulate 65% success + 10% permanent failure
 62        std::uniform_real_distribution<double> dist(0.0, 1.0);
 63        double v = dist(rng);
 64        if (v < 0.65) {
 65            return {PaymentStatus::SUCCESS, "RAZOR-" + req.id + "-" + std::to_string((int)(v*10000)), "OK"};
 66        } else if (v < 0.75) {
 67            return {PaymentStatus::FAILED, "", "Razorpay: permanent rejection (insufficient funds)"};
 68        } else {
 69            return {PaymentStatus::FAILED, "", "Razorpay: transient network issue"};
 70        }
 71    }
 72    string name() const override { return "razorpay"; }
 73};

 75/* -------------------- Retry Strategy Pattern -------------------- */
 76
 77class RetryStrategy {
 78public:
 79    virtual bool shouldRetry(int attempt, const PaymentResponse &lastResponse) = 0;
 80    virtual std::chrono::milliseconds nextDelay(int attempt) = 0;
 81    virtual ~RetryStrategy() = default;
 82};
 83
 84class ImmediateRetryStrategy : public RetryStrategy {
 85    int maxAttempts;
 86public:
 87    ImmediateRetryStrategy(int maxAttempts_ = 3) : maxAttempts(maxAttempts_) {}
 88    bool shouldRetry(int attempt, const PaymentResponse &lastResponse) override {
 89        return attempt < maxAttempts;
 90    }
 91    chrono::milliseconds nextDelay(int attempt) override {
 92        return 0ms;
 93    }
 94};
 95
 96class ScheduledRetryStrategy : public RetryStrategy {
 97    int maxAttempts;
 98    chrono::milliseconds interval;
 99public:
100    ScheduledRetryStrategy(int maxAttempts_, chrono::milliseconds interval_) : maxAttempts(maxAttempts_), interval(interval_) {}
101    bool shouldRetry(int attempt, const PaymentResponse &lastResponse) override {
102        return attempt < maxAttempts;
103    }
104    chrono::milliseconds nextDelay(int attempt) override {
105        return interval;
106    }
107};
108
109class ExponentialBackoffStrategy : public RetryStrategy {
110    int maxAttempts;
111    chrono::milliseconds baseDelay;
112public:
113    ExponentialBackoffStrategy(int maxAttempts_, chrono::milliseconds baseDelay_ = 200ms) : maxAttempts(maxAttempts_), baseDelay(baseDelay_) {}
114    bool shouldRetry(int attempt, const PaymentResponse &lastResponse) override {
115        return attempt < maxAttempts;
116    }
117    chrono::milliseconds nextDelay(int attempt) override {
118        // attempt starts from 1
119        long long factor = 1LL << (attempt - 1);
120        auto ms = baseDelay.count() * factor;
121        if (ms > 5000) ms = 5000; // cap 5s for demo
122        return chrono::milliseconds(ms);
123    }
124};

128class NotificationService {
129public:
130    void notifySuccess(const PaymentRequest &req, const PaymentResponse &resp) {
131        cout << "[Notification] Payment " << req.id << " succeeded via " << req.provider
132             << " tx=" << resp.providerTxId << " amount=" << req.amount << " " << req.currency << "\n";

140/* -------------------- Payment Service (Standardized flow) -------------------- */
141
142class PaymentService {
143    unordered_map<string, shared_ptr<PaymentGateway>> providers;
144    shared_ptr<RetryStrategy> retryStrategy;
145    NotificationService notifier;
146
147public:
148    PaymentService(shared_ptr<RetryStrategy> strat) : retryStrategy(strat) {}
149
150    void registerProvider(shared_ptr<PaymentGateway> gateway) {
151        providers[gateway->name()] = gateway;
152    }
153
154    // Basic validation
155    bool validate(const PaymentRequest &req, string &err) {
156        if (req.amount <= 0) { err = "amount must be > 0"; return false; }
157        if (req.currency.empty()) { err = "currency missing"; return false; }
158        if (req.userId.empty()) { err = "userId missing"; return false; }
159        if (providers.find(req.provider) == providers.end()) { err = "unknown provider"; return false; }
160        return true;
161    }
162
163    PaymentResponse executeWithRetries(const PaymentRequest &req) {
164        auto gw = providers.at(req.provider);
165        int attempt = 0;
166        PaymentResponse lastResp{PaymentStatus::PENDING, "", "not attempted"};
167        while (true) {
168            attempt++;
169            cout << "[PaymentService] Attempt " << attempt << " for payment " << req.id << " via " << gw->name() << "\n";
170
171            // Call provider
172            lastResp = gw->process(req);
173
174            if (lastResp.status == PaymentStatus::SUCCESS) {
175                cout << "[PaymentService] Provider returned SUCCESS on attempt " << attempt << "\n";
176                return lastResp;
177            }
178
179            // Provider returned FAILED - consult retry strategy
180            if (!retryStrategy->shouldRetry(attempt, lastResp)) {
181                cout << "[PaymentService] Retry exhausted after attempt " << attempt << ". Last error: " << lastResp.message << "\n";
182                return lastResp;
183            }
184
185            auto delay = retryStrategy->nextDelay(attempt);
186            cout << "[PaymentService] Will retry after " << delay.count() << "ms\n";
187            if (delay.count() > 0) std::this_thread::sleep_for(delay);
188        }
189    }
190
191    // High-level standardized flow for a single payment request
192    void processPayment(PaymentRequest req) {
193        cout << "=== Processing Payment id=" << req.id << " provider=" << req.provider << " amount=" << req.amount << " ===\n";
194        string err;
195        if (!validate(req, err)) {
196            cout << "[PaymentService] Validation failed: " << err << "\n";
197            PaymentResponse resp{PaymentStatus::FAILED, "", "validation error: " + err};
198            notifier.notifyFailure(req, resp);
199            return;
200        }
201
202        PaymentResponse resp = executeWithRetries(req);
203
204        if (resp.status == PaymentStatus::SUCCESS) {
205            // Update status (DB write would happen here)
206            notifier.notifySuccess(req, resp);
207        } else {
208            notifier.notifyFailure(req, resp);
209        }
210    }
211};

213/* -------------------- Recurring Scheduler (simple demo) -------------------- */
214
215class RecurringScheduler {
216    PaymentService &svc;
217    std::atomic<bool> running { true };
218    vector<PaymentRequest> scheduled; // shallow - copies are fine for demo
219    mutex mtx;
220    thread worker;
221
222public:
223    RecurringScheduler(PaymentService &svc_) : svc(svc_) {}
224
225    ~RecurringScheduler() {
226        stop();
227    }
228
229    void start() {
230        worker = thread([this]() { this->runLoop(); });
231    }
232
233    void stop() {
234        running = false;
235        if (worker.joinable()) worker.join();
236    }
237
238    // schedule a recurring payment: cycles = number of times including initial
239    void scheduleRecurring(const PaymentRequest &req) {
240        lock_guard<mutex> lk(mtx);
241        scheduled.push_back(req);
242    }
243
244private:
245    void runLoop() {
246        // For demo: we'll run limited cycles and short intervals.
247        while (running) {
248            vector<PaymentRequest> toRun;
249            {
250                lock_guard<mutex> lk(mtx);
251                toRun = scheduled; // copy
252                // remove those with 0 cycles after picking them; update remainingCycles
253                scheduled.clear();
254                for (auto &r : toRun) {
255                    if (r.remainingCycles > 0) r.remainingCycles--;
256                }
257            }
258
259            if (!toRun.empty()) {
260                for (auto &r : toRun) {
261                    // Process one cycle
262                    svc.processPayment(r);
263
264                    // If still cycles left, re-schedule after a short demo interval
265                    if (r.remainingCycles > 0) {
266                        // in real system we'd compute next execution time and persist; here we re-add to scheduled
267                        lock_guard<mutex> lk(mtx);
268                        scheduled.push_back(r);
269                    } else {
270                        cout << "[Scheduler] Completed recurring cycles for payment " << r.id << "\n";
271                    }
272                }
273            }
274
275            // Demo interval between scheduler ticks
276            std::this_thread::sleep_for(700ms);
277
278            // Stopping condition for demo: when no scheduled tasks remain
279            {
280                lock_guard<mutex> lk(mtx);
281                if (scheduled.empty()) {
282                    // Wait one more tick and then stop automatically (for demo)
283                    std::this_thread::sleep_for(300ms);
284                    running = false;
285                }
286            }
287        }
288        cout << "[Scheduler] Exiting scheduler loop\n";
289    }
290};

292/* -------------------- Demo / Example usage -------------------- */
293
294int main() {
295    ios::sync_with_stdio(false);
296    cin.tie(nullptr);
297
298    // Choose retry strategy for PaymentService
299    // You can swap between ImmediateRetryStrategy, ScheduledRetryStrategy, ExponentialBackoffStrategy
300    auto retryStrategy = make_shared<ExponentialBackoffStrategy>(4, 150ms);
301    PaymentService paymentService(retryStrategy);
302
303    // Register providers
304    paymentService.registerProvider(make_shared<PaytmAdapter>());
305    paymentService.registerProvider(make_shared<RazorpayAdapter>());
306
307    // Non-recurring example
308    PaymentRequest p1;
309    p1.id = "txn-1001";
310    p1.provider = "paytm";
311    p1.amount = 199.99;
312    p1.currency = "INR";
313    p1.userId = "user-42";
314    p1.recurring = false;
315
316    paymentService.processPayment(p1);
317
318    // Recurring example: 3 cycles total (initial + 2 more)
319    PaymentRequest rp;
320    rp.id = "rec-2001";
321    rp.provider = "razorpay";
322    rp.amount = 49.99;
323    rp.currency = "INR";
324    rp.userId = "user-77";
325    rp.recurring = true;
326    rp.remainingCycles = 3; // number of times to attempt for demo
327
328    RecurringScheduler scheduler(paymentService);
329    scheduler.scheduleRecurring(rp);
330    scheduler.start();
331
332    // Wait for scheduler to finish its demo run
333    while (true) {
334        // Keep main alive until scheduler exits (demo)
335        std::this_thread::sleep_for(200ms);
336        break;
337    }
338
339    // join and cleanup
340    scheduler.stop();
341
342    cout << "=== Demo finished ===\n";
343    return 0;
344}
